  4. 검색 기능 리팩토링 (src/services/aiToolsService.js)

  목표: Postgres FTS(Full Text Search) 대신 동의어 질의 확장, ilike, 다중 컬럼 OR 조건을 활용한 검색 방식으로 교체하여 한국어 검색 정확도를
  높입니다.

  수정 내용:
   * 기존 `textSearch` 제거: AIToolsService 클래스의 searchTools 메서드에서 기존의 textSearch 기반 검색 로직을 제거했습니다.
   * 동의어 사전 및 질의 확장 함수 추가:
       * 간단한 한국어 동의어 사전 SYN 객체를 정의했습니다.
       * 사용자 질의를 동의어와 함께 확장하는 expandQuery 함수를 추가했습니다.
   * 새로운 `searchTools` 구현:
       * searchTools 함수를 AIToolsService 클래스 외부로 이동하여 독립적인 export 함수로 만들었습니다.
       * expandQuery 함수를 통해 확장된 검색어를 기반으로 name, description, category, subcategory 필드에 대해 ilike 및 .or() 필터를 적용하는
         새로운 검색 로직을 구현했습니다.
       * 검색어가 비어있을 경우 기본 목록(최대 50개)을 반환하도록 처리했습니다.

  해결된 문제:
   * 한국어 검색 시 "보고서"와 "문서 작성"처럼 유사한 의미의 단어를 제대로 인식하지 못하던 문제.
   * Postgres FTS의 CJK(중국어, 일본어, 한국어) 토크나이저의 한계로 인한 검색 정확도 문제.

  5. 오류 처리 및 디버깅

  작업 과정에서 발생했던 주요 오류들은 다음과 같으며, 모두 해결했습니다:

     Lucide.Star로 변경하여 수정했습니다.
   * `searchTools` 함수 위치 오류: src/services/aiToolsService.js에서 searchTools 함수가 클래스 내부에 잘못 정의되어 발생한 문제를 클래스
     외부로 이동하여 해결했습니다.
